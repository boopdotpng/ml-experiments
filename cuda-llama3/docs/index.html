One model.
Two regimes.

Prefill has a matrix of tokens and rewards tensor-core reuse. Decode has one token and is dominated by streaming the model weights. Using the same projection kernel for both loses.

Prefill = matrix × matrix

Many prompt rows reuse each weight tile. WMMA maps 16×16 BF16 fragments onto hardware MMA and accumulates in FP32.

Y[M,N] = X[M,K] · W[N,K]ᵀ
tile: 64 rows × 128 columns
one B fragment → four A fragments

Decode = matrix × vector

Only one useful row of activations exists. Four output rows per warp reuse each loaded input and issue coalesced weight reads.

warp → 4 output rows
lane → columns lane, lane+32, ...
4 FP32 sums → shuffle reduce → BF16

FlashAttention,
unpacked.

FlashAttention is not an approximation. It is exact softmax reorganized so scores stay on chip. Step through one scalar value channel below; the production kernel does 64 channels across a warp and splits decode history into eight stripes.

online softmax simulator

When a score beats the running maximum, every earlier contribution is rescaled into the new coordinate system by exp(old m − new m).

The optimization
ledger.

The fast path is mostly about removing traffic, duplicated work, and launch boundaries while leaving the numerical operations recognizable.

01

BF16 cache

K/V traffic and capacity are halved versus the FP32 HIP reference. Dot products and value sums still accumulate in FP32.

02

Segmented GEMM

Q/K/V and gate/up are logical segments in one grid. We save launches without repacking or hiding weight layouts.

03

Fused epilogues

Residual adds and SiLU happen as projection sums leave registers. Intermediates do not take a round trip through memory.

04

RMS scale only

Decode reductions write one FP32 scalar. Projection lanes apply scale and norm weight while loading the residual vector.

05

Split-context FA

Eight independent online-softmax states expose history parallelism and merge stably. At 512 context, decode rose from ~238 to 374 tok/s.

06

CUDA Graph

One graph launch replaces the CPU dispatch of the complete token pipeline. Argmax and generation state remain on device.
